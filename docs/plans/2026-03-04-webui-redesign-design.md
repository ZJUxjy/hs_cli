# Web UI 重构设计文档

## 概述

本设计文档描述了炉石传说 CLI 游戏 Web UI 的全面重构方案，目标是创建一个具有神秘奇幻风格、接近炉石官方视觉效果的现代响应式界面。

## 设计目标

- **美学风格**: 神秘奇幻风 - 深邃紫蓝 + 金色魔法粒子 + 符文装饰
- **技术方案**: 纯 CSS 动画（不引入框架）
- **设计参考**: 接近炉石官方风格
- **适配范围**: 响应式设计（移动端 + PC 端）

## 界面设计

### 1. 主菜单

**视觉效果:**
- 深色虚空渐变背景
- 缓慢漂浮的魔法粒子（金色/蓝色光点）
- 远处星光点缀
- 标题使用 Cinzel 字体，金色渐变 + 外发光效果
- 按钮具有金属质感边框，悬浮时发光 + 微微上浮
- 语言选择悬浮在右上角，半透明背景
- 页面加载时元素依次淡入（交错动画）

**关键动画:**
```css
@keyframes float-particle {
  0%, 100% { transform: translateY(0) translateX(0); opacity: 0.3; }
  50% { transform: translateY(-20px) translateX(10px); opacity: 0.8; }
}

@keyframes title-glow {
  0%, 100% { text-shadow: 0 0 20px rgba(240, 182, 52, 0.5); }
  50% { text-shadow: 0 0 40px rgba(240, 182, 52, 0.8), 0 0 60px rgba(240, 182, 52, 0.4); }
}
```

### 2. 卡牌设计

**卡牌结构:**
- 费用水晶（左上角，蓝色圆形）
- 稀有度宝石（右上角，水滴形状）
- 卡牌插画区域（预留位置）
- 卡牌名称（金色发光字体）
- 卡牌类型（随从/法术/武器）
- 卡牌描述（斜体魔法文字）
- 攻击力/生命值（底部左右角）

**稀有度边框:**
| 稀有度 | 边框样式 | 发光效果 |
|--------|----------|----------|
| 普通 | 银灰色金属边框 | 无 |
| 稀有 | 蓝色水晶边框 | 淡蓝色微光 |
| 史诗 | 紫色奥术边框 | 紫色脉冲光 |
| 传说 | 金色龙鳞边框 + 橙色龙眼 | 强烈金色脉动光 |

**交互效果:**
```css
.card:hover {
  transform: translateY(-10px) scale(1.05);
  box-shadow: 0 20px 40px rgba(0, 0, 0, 0.4), 0 0 30px rgba(240, 182, 52, 0.3);
}

@keyframes legendary-breathe {
  0%, 100% { box-shadow: 0 0 20px rgba(240, 182, 52, 0.4); }
  50% { box-shadow: 0 0 40px rgba(240, 182, 52, 0.8), 0 0 60px rgba(255, 140, 0, 0.4); }
}
```

**手牌布局:**
- 扇形排列，微微倾斜
- 悬浮时卡牌上移并放大

### 3. 战场界面

**布局结构:**
```
┌────────────────────────────────────────────────────────────┐
│  [回合指示器]              [敌方英雄]              [法力]  │
├────────────────────────────────────────────────────────────┤
│    [敌方随从区] (红色边框)                                │
│  ═════════════════ 中线能量波动 ════════════════════════  │
│    [我方随从区] (绿色边框)                                │
├────────────────────────────────────────────────────────────┤
│  [玩家英雄]                                               │
├────────────────────────────────────────────────────────────┤
│    [手牌扇形]                                             │
├────────────────────────────────────────────────────────────┤
│    [控制按钮]                                             │
└────────────────────────────────────────────────────────────┘
```

**背景效果:**
- 深邃虚空渐变 + 暗色符文纹理
- 中线处有缓慢流动的魔法能量
- 少量漂浮的魔法光点
- 四角有淡金色符文边框

**英雄头像:**
- 圆形头像，80x80px
- 金属边框（金色渐变）
- 职业色彩光环（box-shadow）
- 敌方红色边框，我方金色边框

**随从设计:**
- 倾斜的六边形风格边框（clip-path）
- 渐变背景
- 可攻击状态：绿色边框 + 脉冲光效
- 嘲讽随从：顶部金色标签

### 4. 战斗动效

**攻击特效:**
- 攻击线：从攻击者到目标划过一道能量线
- 命中闪光：目标处白色闪光 + 屏幕轻微震动
- 伤害数字：红色数字从目标弹出并上浮消失

```css
@keyframes damage-pop {
  0% { transform: translateY(0) scale(0.5); opacity: 0; }
  20% { transform: translateY(-10px) scale(1.2); opacity: 1; }
  100% { transform: translateY(-50px) scale(1); opacity: 0; }
}

@keyframes screen-shake {
  0%, 100% { transform: translateX(0); }
  25% { transform: translateX(-5px); }
  75% { transform: translateX(5px); }
}
```

**回合过渡:**
- 大字显示 "你的回合" 或 "敌方回合"
- 从模糊放大到清晰再到消失

**抽卡动画:**
- 卡牌从牌堆翻转飞入手牌区域

**法术释放:**
- 卡牌放大发光然后消失

### 5. 响应式设计

**断点定义:**
```css
/* 移动端 - 手机竖屏 */
@media (max-width: 576px) { }

/* 平板 - 手机横屏/小平板 */
@media (min-width: 577px) and (max-width: 768px) { }

/* 桌面 - 大平板/小桌面 */
@media (min-width: 769px) and (max-width: 1024px) { }

/* 大桌面 */
@media (min-width: 1025px) { }
```

**移动端适配策略:**
| 界面 | 移动端适配 |
|------|------------|
| 主菜单 | 垂直堆叠按钮，缩小标题字体 |
| 卡牌 | 单列滚动，点击放大查看详情 |
| 战场 | 垂直布局，手牌底部收起，点击展开 |
| 卡组构建 | 上下分栏，卡牌池和卡组切换显示 |

**移动端战场布局:**
- 顶部信息栏（回合、英雄血量、法力）
- 敌方区域（英雄 + 随从）
- 我方区域（随从 + 英雄）
- 控制按钮
- 手牌（收起状态，点击展开）

## 实施范围

1. **HTML 重构** - 优化语义化结构，添加必要的动画容器
2. **CSS 重写** - 全新的设计系统和组件样式
3. **动画实现** - 所有定义的 CSS 动画和过渡效果
4. **响应式适配** - 四个断点的完整适配
