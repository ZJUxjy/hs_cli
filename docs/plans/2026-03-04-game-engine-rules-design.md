# 炉石传说游戏引擎修改设计方案

> 本设计基于炉石传说官方规则（hearthstone.fandom.com/wiki/Hearthstone_Wiki）及公开游戏资料

## 1. 项目背景与问题分析

### 1.1 当前问题
1. **卡牌类型不匹配**：代码使用小写 `type === 'minion'`，数据使用大写 `type === 'MINION'`
2. **UI显示不全**：法力值、剩余卡牌数量、战场随从信息显示不完整
3. **游戏机制不完整**：缺少部分回合阶段处理和卡牌效果触发逻辑

### 1.2 卡牌数据现状
- 数据量充足：每职业 1000+ 张可收集卡牌
- 数据结构正确：包含 attack, health, cost, type 等字段
- 核心问题：类型判断逻辑错误

## 2. 游戏核心规则（基于真实炉石传说）

### 2.1 游戏开始
- **先手**：抽4张牌（3+1），可换牌
- **后手**：抽5张牌（4+1），获得幸运币（+1临时法力水晶），可换牌
- **初始法力水晶**：1个

### 2.2 回合流程（5个阶段）

```
阶段1: 回合开始 → 阶段2: 法力补充 → 阶段3: 抽牌 → 阶段4: 主阶段 → 阶段5: 结束阶段
```

#### 阶段1: 回合开始
- 触发"回合开始时"效果
- 休眠随从唤醒
- 冻结状态解除

#### 阶段2: 法力补充
- 法力水晶上限+1（最大10）
- 所有法力水晶重置为满
- 过载水晶不解锁

#### 阶段3: 抽牌
- 抽1张牌
- 触发"抽牌时"效果
- 牌库为空时触发疲劳伤害

#### 阶段4: 主阶段
- 召唤随从（战场最多7个）
- 使用法术
- 装备武器
- 随从攻击
- 使用英雄技能

#### 阶段5: 结束阶段
- 触发"回合结束时"效果
- 未使用的临时法力水晶销毁

### 2.3 核心机制

#### 已实现（需修正描述）
| 机制 | 正确说明 |
|------|----------|
| 战吼 (Battlecry) | 进场时立即触发 |
| 亡语 (Deathrattle) | 死亡时触发 |
| 冲锋 (Charge) | 召唤当回合可攻击，但必须指定敌方随从或英雄 |
| 休眠 (Dormant) | 特定条件满足时唤醒（如使用X张卡后） |
| 冻结 (Freeze) | 下一回合无法攻击 |
| 嘲讽 (Taunt) | 必须优先攻击 |
| 圣盾 (Divine Shield) | 抵挡**敌方来源**的第一次伤害 |
| 潜行 (Stealth) | 无法被选中 |
| 剧毒 (Poisonous) | 击杀任何随从 |
| 吸血 (Lifesteal) | 伤害转化为生命 |
| 过载 (Overload) | 下回合阶段1解锁水晶 |
| 法术爆发 (Spellburst) | 施放法术后触发 |

#### 已实现（完整）
| 机制 | 说明 |
|------|------|
| 激励 (Inspire) | 使用英雄技能时触发 |
| 回响 (Echo) | 回合结束时获得复制 |
| 腐蚀 (Corrupt) | 使用更强卡牌时触发 |
| 复生 (Reborn) | 亡语后以1血复活 |
| 磁力 (Magnetic) |  attach to other mechs |
| 奥秘 (Secret) | 敌方触发时生效 |
| 任务/任务线 (Quest/Questline) | 满足条件获得奖励 |

## 3. 修改方案

### 3.1 核心逻辑修复（Critical - 优先修复）

#### 3.1.1 卡牌类型统一（Critical Bug）
```javascript
// 问题：代码使用小写 'minion'，数据使用大写 'MINION'
// 影响：导致卡牌效果无法触发

// 修复方案：统一使用大小写不敏感比较
// 工具函数 src/utils/cardUtils.js
const CardType = {
  isMinion: (card) => card.type === 'MINION' || card.type === 'minion',
  isSpell: (card) => card.type === 'SPELL' || card.type === 'spell',
  isWeapon: (card) => card.type === 'WEAPON' || card.type === 'weapon',
  isHero: (card) => card.type === 'HERO' || card.type === 'hero'
};
```

**涉及文件**（需修复）：
- `src/game/GameEngine.js` - 第349行, 351行
- `src/game/TurnManager.js` - 第153行
- `src/game/AIEngine.js` - 第85, 102, 182, 188, 372行
- `src/game/DiscoverPool.js` - 第23, 26, 29行
- `src/game/RuleEngine.js` - 第30行

#### 3.1.2 游戏阶段完善
在 `TurnManager.js` 中实现完整的5阶段处理：
- `startTurnPhase()` - 回合开始：触发回合开始效果、唤醒休眠、解除冻结
- `refillManaPhase()` - 法力补充：水晶+1（最大10），重置满，过载水晶**在下回合阶段1解锁**
- `drawCardPhase()` - 抽牌：抽1张，触发抽牌效果，牌库为空时疲劳伤害
- `mainPhase()` - 主阶段
- `endTurnPhase()` - 结束阶段：触发结束效果，销毁临时法力水晶

#### 3.1.3 遗漏机制补充
- **幸运币**：后手获得+1临时法力水晶
- **过载**：过载水晶在**下回合阶段1**解锁（而非本回合不解锁）
- **临时法力水晶**：回合结束时销毁

#### 3.1.4 代码重复修复
- 删除 `GameEngine.js` 中重复的 `checkDormantWakeup()` 和 `executeWakeEffect()` 方法
