# 多语言支持设计文档

> **For Claude:** 实现前请先创建 implementation plan

## 目标

为炉石传说 CLI 游戏添加多语言支持，支持中文、英文、日文等多种语言，实现启动时语言选择和运行时文本切换。

## 架构

### 文件结构

```
src/i18n/
  index.js           # i18n 核心模块
  locales/
    zh.json          # 中文翻译
    en.json          # 英文翻译
    ja.json          # 日文翻译
    ko.json          # 韩文翻译
```

### 核心模块设计

#### I18n 类 (index.js)

```javascript
class I18n {
  constructor() {
    this.currentLocale = 'zh';
    this.translations = {};
    this.fallbackLocale = 'en';
  }

  // 初始化 - 加载语言文件
  async init(locale);

  // 切换语言
  async setLocale(locale);

  // 翻译函数 t('key.subkey')
  t(key, params?: object): string;

  // 获取当前语言
  getLocale(): string;

  // 获取支持的语言列表
  getSupportedLocales(): array;
}
```

## 实现策略

### 1. 分层翻译键

使用点号分隔的键名，支持嵌套：
- `ui.menu.*` - 菜单界面
- `ui.game.*` - 游戏界面
- `mechanic.*` - 机制名称
- `class.*` - 职业名称
- `card.*` - 卡牌翻译（从卡牌数据中引用）

### 2. 后端翻译支持

- 后端日志使用英文（避免调试困难）
- 游戏消息（setMessage）使用翻译键
- API 返回游戏状态时附带翻译

### 3. 前端翻译集成

- 页面加载时初始化 i18n
- UI 文本通过 `i18n.t()` 渲染
- 卡牌描述动态翻译

### 4. 语言选择

- 主菜单添加语言选择入口
- 保存选择到 localStorage
- 首次启动默认中文

## 翻译键命名规范

| 类别 | 前缀 | 示例 |
|------|------|------|
| 菜单 | ui.menu. | ui.menu.start |
| 游戏 | ui.game. | ui.game.turn |
| 按钮 | ui.button. | ui.button.endTurn |
| 消息 | ui.message. | ui.message.victory |
| 机制 | mechanic. | mechanic.taunt |
| 职业 | class. | class.warrior |
| 卡牌 | card.{id}. | card.cs2_101.name |

## 测试计划

1. 验证所有 UI 文本正确显示
2. 切换语言后 UI 立即更新
3. 卡牌描述正确翻译
4. 机制图标文字正确
5. 语言持久化正确

## 风险与限制

- 卡牌数据中的英文文本可以作为翻译参考
- 部分游戏术语难以翻译（如 "Battlecry"）
- 日文/韩文字符可能需要特殊字体支持
