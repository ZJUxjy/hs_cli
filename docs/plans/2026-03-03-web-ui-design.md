# Web UI 设计文档

> 验证日期: 2026-03-03

## 1. 概述

为炉石传说CLI游戏开发Web UI界面，支持卡组构建和游戏对战。

## 2. 技术架构

### 2.1 前后端分离
- **后端**: Express.js 提供 REST API
- **前端**: 纯 HTML/CSS/JavaScript
- **通信**: HTTP JSON API

### 2.2 目录结构
```
hs_minimax/
├── public/                 # 前端静态文件
│   ├── index.html         # 主页面
│   ├── css/
│   │   └── style.css      # 样式
│   └── js/
│       ├── app.js          # 主应用
│       ├── api.js          # API调用
│       ├── deck.js         # 卡组构建
│       └── game.js         # 游戏界面
├── src/
│   └── ...
└── server.js               # Express服务器
```

## 3. 功能模块

### 3.1 主界面
- 开始游戏按钮
- 卡组管理按钮
- 设置按钮

### 3.2 卡组构建界面
- 职业选择
- 卡牌搜索/筛选（按费用、类型、稀有度）
- 卡牌拖拽到卡组槽位
- 卡组预览（费用分布、职业分布）
- 保存/删除卡组

### 3.3 游戏界面
- 双方英雄信息（生命、护甲、法力）
- 战场（双方随从显示）
- 手牌区
- 回合信息
- 操作按钮（结束回合、使用英雄技能等）

## 4. API 设计

### 4.1 卡组相关
```
GET  /api/decks              # 获取所有卡组
POST /api/decks              # 创建卡组
GET  /api/decks/:id          # 获取指定卡组
PUT  /api/decks/:id          # 更新卡组
DELETE /api/decks/:id        # 删除卡组
```

### 4.2 卡牌相关
```
GET  /api/cards              # 获取所有卡牌
GET  /api/cards/:id         # 获取指定卡牌
GET  /api/cards/class/:cls   # 获取职业卡牌
```

### 4.3 游戏相关
```
POST /api/game/start         # 开始新游戏
POST /api/game/play          # 打出手牌
POST /api/game/attack        # 随从攻击
POST /api/game/endTurn       # 结束回合
GET  /api/game/state         # 获取游戏状态
```

## 5. 界面设计

### 5.1 卡组构建
- 左侧：可用卡牌列表（可搜索、筛选）
- 右侧：卡组槽位（30格）
- 底部：卡组统计（费用曲线、职业分布）

### 5.2 游戏界面
- 顶部：回合信息、消息栏
- 中间：战场（敌方战场 + 玩家战场）
- 底部：玩家手牌、玩家英雄信息
- 右上：敌方信息（隐藏手牌数）

## 6. 交互设计

### 6.1 卡组构建
- 拖拽卡牌到卡组槽位
- 点击卡组中的卡牌可移除
- 双击卡牌可直接添加

### 6.2 游戏
- 点击手牌选中
- 点击战场打出随从
- 点击敌方随从/英雄进行攻击
- 右键取消选择
